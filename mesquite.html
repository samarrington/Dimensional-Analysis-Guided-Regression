Introduction¶

Often all we care about is building a model that can make good predictions. Sometimes, however, we care about explainability of model as well and many popular machine learning algorithms are not particularly explainable. When it comes to physical data (kilograms, meters, coulombs, etc.), we may also care about dimensional soundness. Fortunately, these dimensions have meanings; they aren't arbitrary. Because they have meanings, relationships between variables with such units must follow rules. We can leverage these rules to build a dimensionally sound and explainable model.

Dimensional Analysis¶

The following approach is heavily dependent on an understanding of dimensional analysis. The general idea behind dimensional analysis is that if we are performing operations on quantities that have units, the operations are performed on the units themselves as well. We must then ensure dimensional equality as well as quantitative. For example, let's take a look at the formula for volumetric density of a cylinder.

$$ \rho= \dfrac{m}{\pi r^{2}h} $$

*For the following dimensional analysis, we'll assume generic units. Radius and height will be considered length units and called $L$. Mass will be given a generic label $M$. $\pi$ is a dimensionless constant.

From this we can gather the units of density ($\rho$) given the equation. To avoid confusion, I'll represent "has units" as $:$.

$$ \rho : \dfrac{M}{L*L*L}:M/L^{3} $$

We can also use this idea "check" relationships. For a given equation to be dimensionally sound, both sides of the equation must be consistent dimensionally.

Note: When working with data that includes actual units, it makes sense to work with the same unit across all of such unit type (ex. use mm for all length units). This won't actually improve the models predictive ability, as unit conversions will be baked into fitting coefficients, but it will make things "cleaner".

Guessing Relationships¶

Just a heads up: we don't actually use this process in the example with an actual dataset. Nonetheless, it is useful to know and in the same vein as the rest of the work here, so I figured I would add it.

This idea can also help us guess relationships. If it's simple, we can usually guess the relationship quickly and confirm dimensional consistency. For more complex relationships, Rayleigh's method of dimensional analysis helps. I'll outline the idea using the data science terminology of "target" and "features".

Let $F_{1}, F_{2}, ... F_{n}$ be the set of $n$ predictive features. Let $T$ be the target. *The goal is to find:

$$ T = CF_{1}^{m_1}F_{2}^{m_2} \cdots F_{n}^{m_n} $$

Given that we know the dimensions of each feature, $F_i$, and the dimensions of the target, $T$, we can solve this relationship dimensionally by substituting in units and breaking the problem down into $l$ linear equations, where $l$ is determined to be the number of unique base units in $F_{1}, F_{2}, ... F_{n}$. I'll use volumetric density of a cylinder for an example.

*Note: We can also look at summing terms as well. As long as each individual term matches the dimensions on the left side of the equation. We will actually do this later in the main example.

Example¶

I'm going to use the volumetric density of a cylinder as an example here. It's simple and we know the solution so keep in mind that for more difficult problems, more work and some linear algebra will be necessary.

The target is the density $\rho$ and the features are $m$, $r$, and $h$. We'll assume the same base units as before for each of these values. So now we have that:

$$ \rho = Cm^ar^bh^c $$

Using the base units of each, we can set up a single equation representing dimensional consistency. It's best to keep out fractions and instead use negative exponents to reduce errors for the next step. We can also ignore the constant, $C$, for the time being as we are assuming a dimensionless constant.

$$ M^1L^{-3}=M^aL^bL^c $$

We now break this down into several linear equations.

$$ 1 = a $$$$ -3 = b+c $$

The first equation is already solved for us and the second has an infinite number of solutions. This will not always be the case. Often there will be a single solution and other times no solution exists (which means you have to take a look at unit assumptions and ask yourself if such a relationships exists). For more complex relationships, linear algebra will expedite the process.

Enough about other possibilities though; let's continue to look at this specific equation. We know that an infinite number of solutions are possible so we guess and check likely solutions based on domain knowledge and experience. For this example, we understand that the equation represents the "volume" portion of density, and we know from experience that $V \sim Ah$ so we guess that $b=-2$ and $c=-1$.

Now we plug back in this solution to get:

$$ \rho = C \dfrac{m}{r^2 h} $$

So what's the deal with $C$? We know it should be $1/\pi$ but using this technique, we're only accurate to a constant. Remember that our use case is in the data science field, so we can run a regression to determine $C$. Essentially, all we've done thus far is extract the feature $\dfrac{m}{r^2 h}$ which we can run a linear regression on. Next, I'll go over a short example using real data that isn't quite as straight forward and do just this.

Building the Model¶

When building this model, we'll ignore Group for the time being. Based on the description, there isn't a clear logical reason it will significantly help and it certainly won't help when building our dimensionally sound model. It will only distract.

First Theoretical Model¶

Let's quickly define some more equation variables. The target, LeafWeight, has base units of mass $M$, and will be represented as $W_L$ in equations. The two diameters will be represented as $d_1$ and $d_2$. The goal now is to find a dimensionally sound relationship as follows:

$$ W_L = F(d_1, d_2, h_T, h_c, p) $$

The idea is to find some combination of features that represents a volume encompassing a density of leaves. The coefficient extracted in the regression will include the volumetric density of the leaves in that volume.

We start with the knowledge that $V \sim Ah$. We will represent the area as the area of an ellipse, $A_{ellipse} \sim d_1d_2$. We could also add a dimensionless feature, shape, ($s = d_1/d_2$), to represent the asymmetry of the bush. We'll add this in later models. In this attempt, we're also including $p$, which may or may not help. So now we have the model.

$$ W_L = \alpha_0 + \alpha_1 p d_1d_2 h_c $$

$\alpha_0$ must have $M$ units and $\alpha_1$ must have $M/L^{3}$ units for dimensional consistency. This fits our previous description of thinking of $a_1$ as a leaf density in a volume.

This is now our starting model for regression. We'll engineer a new a feature for $p d_1d_2 h_c$ and run a regression with that feature as the only predictive feature.

Now that we've created the new feature, we'll run the linear regression. Instead of running a linear regression a single time, we'll run a bootstrap regression. Essentially, instead of fitting the regression once, the model is fitted on bootstrap samples. The numbers of samples fitted is specified in the samples parameter. The function then outputs a 95% credible interval for the evaluation statistics: root-mean-squared error (sigma), mean-absolute error, and $R^2$. The results for every sample are also returned so that comparisons can be made to other models. Using this method allows us to see a distribution of expected performance, instead of single values. The function below performs this process.

In [11]:
def bootstrap_regression(data, formula, estimator, samples=100):
    
    bootstrap_results = {}
    
    sigmas = []
    r2s = []
    maes = []

    n = data.shape[ 0]
    bootstrap_results[ "n"] = n
    
    
    for i in range(samples):
        
        
        sampling_indices = pd.Index([i for i in [np.random.randint(0, n - 1) for _ in range(0, n)]])
        sample = data.loc[sampling_indices]
        y_sample, X_sample = dmatrices(formula, sample)
        
        estimator.fit(X_sample, y_sample)
        y_predict = estimator.predict(X_sample)
        
        sigma = np.sqrt(mean_squared_error(y_sample, y_predict))
        r2 = r2_score(y_sample, y_predict)
        mae = mean_absolute_error(y_sample, y_predict)
        
        sigmas.append(sigma)
        r2s.append(r2)
        maes.append(mae)
        
    # Get a set of residuals
    y, X = dmatrices(formula, data)
    estimator.fit(X, y)
    y_predict = estimator.predict(X)
    bootstrap_results["residuals"] = y-y_predict
        
    bootstrap_results["resampled_mae"] = pd.Series(maes)
    bootstrap_results["resampled_sigma"] = pd.Series(sigmas)
    bootstrap_results["resampled_r^2"] = pd.Series(r2s)
    

    # Find credible intervals
    mae_cred = np.array2string(stats.mstats.mquantiles(maes, [0.25, .975]), precision=3, separator=',')
    sigma_cred = np.array2string(stats.mstats.mquantiles(sigmas, [0.25, .975]), precision=3, separator=',')
    r2_cred = np.array2string(stats.mstats.mquantiles(r2s, [0.25, .975]), precision=3, separator=',')

    
    # Print results
    print("MAE Resample Mean: {0:.3f}".format(np.mean(maes)))
    print("95% Credible Interval for MAE: ", mae_cred)
    print("Sigma Resample Mean: {0:.3f}".format(np.mean(sigmas)))
    print("95% Credible Interval for sigma: ", sigma_cred)
    print("R2 Resample Mean: {0:.3f}".format(np.mean(r2s)))
    print("95% Credible Interval for R2: ", r2_cred)
    
    
    return bootstrap_results

def power_validation_plot(data, features_to_mult, feature_to_optimize, powers, fit_intercept=True,
                          repetitions=20, folds=5):
    
    scores = {}
    scores['p'] = []
    scores['train'] = []
    scores['train_std'] = []
    scores['test'] = []
    scores['test_std'] = []
    
    # create template feature that is product of all input features other than the one to optimize
    features_to_mult.remove(feature_to_optimize)
    template_feature = None
    for feature in features_to_mult:
        if template_feature is None:
            template_feature = data[feature]
        else:
            template_feature = template_feature*data[feature]
    
    for p in powers:
        
        # create test feature 
        test_feature = template_feature*data[feature_to_optimize]**p
        param_score_train = []
        param_score_test = []
        
        for _ in range(repetitions):
            
            # Perform cross validation and get scores
            kf = KFold(folds, shuffle=True)
            for train_index, test_index in kf.split(test_feature):
                
                test_feature_train = pd.DataFrame(test_feature.iloc[train_index])
                test_feature_test = pd.DataFrame(test_feature.iloc[test_index])
                y_train = pd.DataFrame(data.loc[train_index, 'LeafWeight'])
                y_test = pd.DataFrame(data.loc[test_index, 'LeafWeight'])
                
                # Fit model
                lr = LinearRegression(fit_intercept=fit_intercept)
                lr.fit(test_feature_train, y_train)
                
                # Get scores and add to param_score list
                train_score = lr.score(test_feature_train, y_train)
                test_score = lr.score(test_feature_test, y_test)
                param_score_train.append(train_score)
                param_score_test.append(test_score)
                
        # Append param_scores to 'master' score
        scores['p'].append(p)
        scores['train'].append(np.mean(param_score_train))
        scores['train_std'].append(np.std(param_score_train))
        scores['test'].append(np.mean(param_score_test))
        scores['test_std'].append(np.std(param_score_test))
    
    scores = pd.DataFrame(scores)

    # Getting bounds of confidence interval
    train_upper = scores['train'] + scores['train_std']
    train_lower = scores['train'] - scores['train_std']
    test_upper = scores['test'] + scores['test_std']
    test_lower = scores['test'] - scores['test_std']
    
    # Create plot
    figure = plt.figure(figsize=(12,8))
    axes = figure.add_subplot(1, 1, 1)
    axes.plot(scores["p"], scores["train"], '-', label='train', color ='firebrick')
    axes.plot(scores["p"], scores["test"], '-', label='test', color = 'steelblue')
    axes.fill_between(scores["p"], train_upper, train_lower, color='firebrick', alpha=0.25)
    axes.fill_between(scores["p"], test_upper, test_lower, color='steelblue', alpha=0.25)
    axes.legend()
    axes.set_xlabel('Power')
    axes.set_ylabel(r'$R^2$')
    axes.set_title("Validation Curve for Power of " + feature_to_optimize)
    axes.set_xticks(powers)
    axes.set_ylim([0.0, 1.0])
    
    plt.show()
    plt.close()
    
    return scores

Considering the Intercept¶

At it's current state our model is:

$$ W_L = \alpha_0 + \alpha_1 d_1 d_2 h_c $$

Remember that the goal here is to build a dimensionally sound explainable model. So what do we know about this coefficient $\alpha_0$? We know that it must have $M$ units and we know that it represents LeafWeight ($W_L$) when the either measured diameter of the canopy, or the height of the canopy is $0$. In other words, it's the LeafWeight ($W_L$) without a canopy. Therefore, physically speaking, $\alpha_0$ should be $0$ and removed as a regression coefficient.

Let's see how our model performs when removing $\alpha_0$. The model we are now looking at follows:

$$ W_L = \alpha_1 d_1 d_2 h_c $$

$$ W_L = \alpha_0 + \alpha_1(d_1/d_2)^m d_1 d_2 h_c $$

and optimize $m$.

Adding Group¶

There are certainly more dimensional terms to explore. However, we could in theory endlessly explore options. Therefore, let's leave the dimensional features alone and add in Group. Normally, we would go through some EDA first, but this is not an EDA guide. We'll simply one-hot-encode the variable and add it in as an interacting feature. The model we are now exploring is:

$$ W_L = \alpha_0 + (\alpha_1 + \beta_1)(d_1/d_2)^{1/2} d_1 d_2 h_c + (\alpha_2 + \beta_2) d_1 d_2 h_T $$

The betas are non-zero (value determined by regression) when Group=MCD and 0 otherwise. They represent interaction.

As a side note, we haven't applied cross-validation very much in this project. Normally, we would but this data set is very small ($n=46$) so our test sets would be incredibly small. Looking at the validation curves we did plot, we can see how this can be a problem. The blue area, representing plus or minus one standard deviation, is huge. Also, the goal of the project was to show how to build dimensionally sound, explainable regression models; repeating cross-validations at each step would distract from the goal.

Summary¶

When working with data that includes units with meaning, consider leveraging dimensional analysis. Through dimensional analysis and slowly building a regression model, we can develop a dimensionally sound, highly explainable model. For the mesquite example, we were able to produce a dimensionally sound, explainable model with a mean $R^2$ of 0.93, improving upon scores produced by other approaches on the same dataset.
